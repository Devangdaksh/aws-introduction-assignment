# aws-introduction-assignment

## Q1.What is AWS?
### Ans.AWS (Amazon Web Services) is a massive, secure cloud computing platform provided by Amazon. Instead of a company or student buying physical computers, hard drives, and servers to run software, they rent them from Amazon over the internet.

## Q2.Name one key advantage of using AWS for cloud computing.
### Ans.The Pay-As-You-Go Cost Model: You only pay for the exact amount of digital resources (like storage or computing power) that you use, and only for the time you use them. There are no expensive upfront costs. If you run a virtual computer for two hours to test a school project, you only pay for those two hours, not for the whole month

## Q3.Explain the concept of Elastic Compute Cloud (EC2) in AWS.
### Ans.Amazon EC2 lets you rent virtual computers (called instances) in the cloud. You can use these virtual computers just like your physical laptop to run applications, host websites, or play games.

## Q4.hat are the essential components of AWS CloudFormation?
### Ans.AWS CloudFormation is an automation tool that lets you build your entire cloud setup using code. The three essential components are:
Templates: A text file written in JSON or YAML format that acts as a blueprint or instruction manual (e.g., "I want one virtual computer and one database").  
Stacks: The actual group of live resources that AWS builds for you when it reads your template. If you delete the stack, everything built by that blueprint is cleaned up safely.  
Change Sets: A "preview" button that shows you exactly what changes CloudFormation will make to your live stack before you hit execute, preventing accidental mistakes.  
## Q5.Explain the concept of load balancing in the context of AWS services.
### Ans.In AWS, load balancing is handled by Elastic Load Balancing (ELB). It acts like a digital traffic cop sitting in front of your servers.  When thousands of users try to access your website at the same time, the load balancer evenly splits up the incoming traffic and sends it to different servers so that no single server crashes from being overloaded.
